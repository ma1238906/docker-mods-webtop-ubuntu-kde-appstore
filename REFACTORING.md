# 项目结构重构说明

## 重构概述

本次重构将项目从单一目录结构调整为客户端（client）和服务器（server）分离的架构，为后续开发软件商店服务器功能做准备。

## 目录结构变化

### 重构前

```
root/app/appstore/
├── app/
├── assets/
├── scripts/
├── server/
└── web/
```

### 重构后

```
.
├── client/              # 软件商店客户端（运行在Ubuntu系统中）
│   ├── app/
│   ├── assets/
│   ├── scripts/
│   ├── server/
│   └── web/
│
└── server/              # 软件商店服务器（提供软件资源）
    ├── api/
    ├── data/
    └── static/
```

## 主要变更

1. **客户端代码迁移**：将 `root/app/appstore/` 下的所有内容复制到 `client/` 目录
2. **服务器目录创建**：创建了 `server/` 目录结构，包含API、数据存储等模块
3. **配置文件更新**：
   - 更新了 `README.md` 说明新的目录结构
   - 更新了 `docker-compose.yml` 以支持客户端和服务器分离
   - 为客户端和服务器分别创建了 `Dockerfile` 和 `requirements.txt`

## 兼容性说明

- `root/` 目录下的旧代码保留，确保现有Docker构建不会立即失败
- 客户端代码路径引用基于相对路径，无需修改
- 建议逐步迁移到新的目录结构

## 后续开发计划

1. **客户端增强**：
   - 添加从服务器下载软件信息的功能
   - 支持动态更新软件列表
   - 支持从服务器下载安装脚本和图标

2. **服务器开发**：
   - 实现软件列表API
   - 实现软件详情API
